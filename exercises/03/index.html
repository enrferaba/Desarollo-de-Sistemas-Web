Versión guía para principiantes

1. Objetivo didáctico

Comprender cómo organizar el entregable del Tema 3 · Ejercicio 03: describimos al equipo de trabajo, definimos el plan de aprendizaje y demostramos el uso de bibliotecas externas en un prototipo estático.

Nota del profesor

«Reutilizad el HTML del ejercicio anterior y añadid las bibliotecas nuevas. No hace falta complicarse, pero sí dejar claro qué problema resuelve cada herramienta».

Recuerda: siempre puedes volver a los apuntes en la carpeta teoria/.

2. Bibliotecas que practicamos

Igual que en clase, las enlazamos mediante CDNs para no instalar nada. Mantén conexión a Internet cuando abras la página.

Biblioteca ¿Para qué la usamos? CDN
Bootstrap 5 Estructurar la cuadrícula, tarjetas y tablas sin escribir CSS desde cero. cdn.jsdelivr.net/npm/bootstrap
Google Fonts Aplicar las tipografías Poppins y Fira Code tal como aparece en los apuntes. fonts.googleapis.com
Animate.css Dar animaciones suaves al entrar el contenido (presentación del plan). cdnjs.cloudflare.com/ajax/libs/animate.css
Bootstrap Icons Incorporar iconos explicativos en encabezados y listas. cdn.jsdelivr.net/npm/bootstrap-icons

3. Estructura base reutilizada

Partimos del header → main → footer del ejercicio 01. A continuación mostramos la versión resumida del HTML base para que la puedas comparar con tus apuntes. 

<header> Presentación del equipo </header>
<main>
  <section> Objetivos y roles </section>
  <section> Plan de aprendizaje </section>
  <section> Recursos y enlaces </section>
</main>
<footer> Contacto y recordatorios </footer>

Sobre este esqueleto añadimos componentes de Bootstrap para maquetar columnas y tarjetas. Esto facilita centrarnos en el contenido pedagógico.

4. Desarrollo paso a paso

  1. Analizar la teoría. Revisa el Tema 3 y localiza el apartado «Exercise 03». Resume los puntos clave (roles del equipo, fases y entregables).
  2. Importar bibliotecas. Copia las etiquetas <link> del bloque anterior en el <head>.
  3. Maquetar el contenido. Usa contenedores .container y el sistema de columnas de Bootstrap para organizar la información en filas.
  4. Dar estilo mínimo. Añade variables CSS en :root como hicimos en clase para definir colores coherentes.
  5. Revisar accesibilidad. Comprueba que las imágenes tengan alt, que los encabezados sigan un orden lógico y que el contraste sea alto.

6. Plan de aprendizaje

Cronograma semanal

  • Lunes: Repaso de teoría y planificación (45 min).
  • Miércoles: Práctica con componentes de Bootstrap (60 min).
  • Viernes: Correcciones y subida al repositorio (30 min).

Evidencias

Para entregar en clase:

  • HTML validado con encabezados ordenados.
  • Lista de bibliotecas y enlaces de referencia.
  • Captura de pantalla del prototipo en escritorio y móvil.

7. Checklist final antes de entregar

  • He enlazado todas las bibliotecas y comprobado que se cargan en el navegador.
  • El contenido explica claramente roles, plan de aprendizaje y bibliotecas como en los apuntes.
  • El código está indentado y con etiquetas semánticas.
  • Guardé el archivo y lo subí al repositorio con un mensaje de commit descriptivo.
